code balance

...if we wish to count lines of code, we should not regard them as "lines produced" but as "lines spent"...
— Dijkstra

This visualization shows the accumulated effects of my changes to the tree.

Each bar represents a commit. Red shows added lines, green removed lines. Each bar is centered at the cumulative added/removed count from my commits: the leftmost bar is centered at 0, and if it net added code, the next bar is raised accordingly.

Commits that added and removed the same number of lines are elided.
